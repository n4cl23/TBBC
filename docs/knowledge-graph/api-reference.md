# API semântica

- `GET /api/relations?entity={id}`
- `GET /api/entity/{id}/relations`
- `GET /api/entity/{id}/timeline`
- `GET /api/entity/{id}/related?depth=1`
- `GET /api/graph`
- `GET /api/search?q=&kind=&role=&kingdom=`
- `GET /api/world?query=protects&subject=frost-kingdom`
- `GET /api/kingdom/{slug}/graph`

Todas as respostas são derivadas de `/api/canon`; nenhum endpoint mantém dados próprios.
